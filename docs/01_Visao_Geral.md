# Akeen Commerce Data Platform (ACDP)

## Introdução

O **Akeen Commerce Data Platform (ACDP)** é um projeto pessoal desenvolvido para simular a construção de uma plataforma moderna de dados utilizada por empresas de médio e grande porte.

O projeto representa a plataforma de dados da empresa fictícia **Akeen Commerce**, um e-commerce criado exclusivamente para fins de estudo, experimentação e desenvolvimento profissional.

Mais do que um projeto de programação, o ACDP busca reproduzir os desafios encontrados por equipes reais de Engenharia de Dados, desde a ingestão dos dados até sua disponibilização para análises, dashboards e aplicações baseadas em Inteligência Artificial.

Toda a evolução da plataforma será guiada por boas práticas de arquitetura, documentação, governança, qualidade de dados e escalabilidade.

---

# Propósito

O principal objetivo deste projeto é construir experiência prática nas tecnologias e nos conceitos utilizados por Engenheiros de Dados modernos.

Ao longo do desenvolvimento serão explorados aspectos técnicos e arquiteturais normalmente encontrados em ambientes corporativos, permitindo compreender não apenas como implementar soluções, mas também como tomar decisões de arquitetura, documentá-las e mantê-las ao longo do tempo.

O projeto será tratado como um produto de software em evolução contínua, e não como um exercício isolado.

---

# Objetivos

Os principais objetivos do projeto são:

- Desenvolver experiência prática em Engenharia de Dados moderna.
- Simular uma plataforma corporativa baseada em arquitetura Lakehouse.
- Consolidar conhecimentos em Apache Spark, Databricks e Delta Lake.
- Aprender Analytics Engineering utilizando dbt.
- Construir pipelines escaláveis e reutilizáveis.
- Aplicar boas práticas de DataOps e Engenharia de Software.
- Criar uma base sólida para integração com soluções de Inteligência Artificial.
- Construir um portfólio técnico que represente situações próximas às encontradas no mercado.

---

# Escopo

O projeto contempla todas as principais etapas do ciclo de vida dos dados.

Entre elas:

- Ingestão de dados provenientes de diferentes fontes.
- Armazenamento em Data Lake.
- Transformações utilizando Apache Spark.
- Organização em camadas Bronze, Silver e Gold.
- Modelagem dimensional.
- Analytics Engineering com dbt.
- Consumo através de dashboards.
- Aplicação de testes e validações.
- Monitoramento e observabilidade.
- Documentação técnica.
- Versionamento.
- Evolução contínua da arquitetura.

Futuramente o projeto será expandido para incluir funcionalidades como DataOps, Streaming, Machine Learning e Inteligência Artificial.

---

# Empresa Fictícia

A plataforma representa a empresa fictícia **Akeen Commerce**, um e-commerce que comercializa produtos para consumidores finais.

O ambiente será composto por diferentes domínios de negócio, permitindo simular cenários reais de integração entre áreas da empresa.

Os principais domínios inicialmente previstos são:

- Clientes
- Produtos
- Categorias
- Pedidos
- Itens de Pedido
- Pagamentos
- Entregas
- Estoque
- Marketing
- Fornecedores

Novos domínios poderão ser adicionados conforme a evolução do projeto.

# Princípios do Projeto

Durante todo o desenvolvimento, as seguintes diretrizes deverão ser respeitadas.

## Aprendizado acima da velocidade

Cada componente será implementado buscando compreender profundamente os conceitos envolvidos, evitando soluções que apenas "funcionem" sem entendimento técnico.

---

## Arquitetura antes da implementação

Antes de escrever código, toda funcionalidade deverá possuir uma definição clara de arquitetura, responsabilidades e impactos sobre a plataforma.

---

## Simplicidade com possibilidade de evolução

As soluções deverão ser simples o suficiente para facilitar o aprendizado, mas estruturadas de forma que possam crescer sem necessidade de grandes refatorações.

---

## Documentação como parte do desenvolvimento

Toda decisão arquitetural relevante deverá ser documentada.

A documentação será tratada como parte integrante do projeto e não como uma atividade opcional.

---

## Qualidade desde o início

Mesmo sendo um projeto de estudos, serão adotadas práticas de qualidade utilizadas em ambientes corporativos.

Exemplos:

- Versionamento
- Organização do código
- Padronização
- Testes
- Data Quality
- Logging
- Monitoramento
- Reutilização de componentes

---

# Filosofia

O objetivo do ACDP não é apenas construir pipelines de dados.

O projeto pretende demonstrar como uma plataforma moderna de Engenharia de Dados é concebida, documentada, implementada e evoluída ao longo do tempo.

Cada etapa será encarada como uma oportunidade para aprender conceitos arquiteturais, entender trade-offs tecnológicos e aplicar boas práticas utilizadas por equipes de dados em empresas reais.

Ao final, espera-se que a plataforma represente um ambiente suficientemente completo para integrar Engenharia de Dados, Analytics Engineering, Business Intelligence, DataOps e Inteligência Artificial em um único ecossistema.
